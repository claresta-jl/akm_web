Apa itu Amazon EC2?

Amazon Elastic Compute Cloud (EC2) adalah layanan komputasi yang menyediakan kapasitas komputasi yang bisa diubah ukurannya di cloud AWS. EC2 memungkinkan pengguna untuk menjalankan server virtual (disebut instance) tanpa harus membeli hardware fisik.

NoteAnalogi Sederhana

Bayangkan EC2 seperti menyewa komputer di warnet. Kamu bisa gunakan sesuai kebutuhan, bayar per jam, dan bisa kapan saja berhenti tanpa khawatir perawatan mesin.

1. Komponen Utama EC2

💻

Instance

Server virtual yang berjalan di infrastruktur AWS. Bisa dihidupkan dan dimatikan kapan saja.

📀

AMI (Amazon Machine Image)

Template yang berisi OS dan konfigurasi software untuk membuat instance baru.

🔑

Key Pair

Sepasang kunci (publik & privat) untuk mengakses instance secara aman via SSH.

🛡️

Security Group

Firewall virtual yang mengontrol traffic masuk dan keluar dari instance EC2.

2. Jenis-Jenis Instance EC2

Setiap tipe instance dirancang untuk kebutuhan yang berbeda:

Famili Contoh Kegunaan
General Purpose t3, m5 Web server, dev/test
Compute Optimized c5, c6g Batch processing, HPC
Memory Optimized r5, x1e Database besar, cache
Storage Optimized i3, d3 Data warehouse, NoSQL
Accelerated Computing p3, g4 Machine learning, GPU

3. Amazon Machine Image (AMI)

AMI adalah blueprint yang digunakan untuk membuat EC2 instance. AMI mencakup:

  • Sistem Operasi (Linux, Windows, macOS)
  • Software yang terinstall
  • Konfigurasi awal
  1. Pilih AMI — Bisa dari AWS Marketplace, Community AMI, atau buat sendiri (custom AMI).
  2. Pilih Tipe Instance — Sesuaikan CPU, RAM, dan spesifikasi lain.
  3. Konfigurasi — Network, IAM Role, User Data, dll.
  4. Storage — Tambahkan EBS volume sesuai kebutuhan.
  5. Security Group — Atur aturan firewall.
  6. Launch — Instance siap digunakan.

4. Model Harga EC2

🕐

On-Demand

Bayar per jam/detik tanpa komitmen. Cocok untuk workload tidak terduga atau testing.

📅

Reserved Instances

Komitmen 1–3 tahun. Diskon hingga 75% dibanding On-Demand. Cocok untuk workload stabil.

💸

Spot Instances

Gunakan kapasitas AWS yang tidak terpakai. Harga bisa hemat hingga 90%, namun bisa dihentikan sewaktu-waktu.

🏢

Dedicated Hosts

Server fisik khusus untuk satu pelanggan. Untuk kebutuhan compliance atau lisensi software tertentu.

5. Penyimpanan EC2

Tipe Karakteristik Use Case
EBS (Elastic Block Store) Persisten, dapat di-detach Database, OS
Instance Store Sementara (hilang saat stop) Buffer, cache, temp
EFS (Elastic File System) Shared, scalable Multi-instance sharing
S3 Object storage Backup, file statis

6. Jaringan & Keamanan

Security Group

Security Group bekerja seperti firewall — hanya mengizinkan traffic yang sesuai aturan:

Jenis Arah Contoh
Inbound Rule Masuk ke instance Port 22 (SSH), Port 80 (HTTP)
Outbound Rule Keluar dari instance Semua traffic (default allow)

7. Lifecycle Instance EC2

Sebuah EC2 instance memiliki beberapa status:

  1. Pending — Instance sedang dipersiapkan setelah diluncurkan.
  2. Running — Instance aktif dan berjalan. Tagihan dihitung.
  3. Stopping / Stopped — Instance dihentikan sementara. EBS tetap ada, IP bisa berubah.
  4. Terminating / Terminated — Instance dihapus permanen. Data Instance Store hilang.
ImportantPerbedaan Stop vs Terminate
  • Stop : Instance dimatikan tapi masih ada, bisa dihidupkan lagi
  • Terminate : Instance dihapus permanen, tidak bisa dipulihkan
